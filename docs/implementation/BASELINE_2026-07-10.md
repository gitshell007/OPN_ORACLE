# Baseline técnico — 2026-07-10

## Estado observado

- Rama `master`, commit base `2ab0e14`, alineada con `origin/master` al iniciar la auditoría.
- El árbol contiene cambios y muchos archivos no versionados del prototipo. Deben preservarse; no se autoriza `reset`, `clean` ni una reestructuración destructiva.
- Frontend en la raíz: Next.js 16.2.10, React 19.2.7 y TypeScript 6.0.3, resueltos por `package-lock.json` v3.
- Runtime local: Node.js 26.5.0, npm 11.17.0 y Python 3.9.6.
- Docker y Docker Compose no están disponibles en este entorno local.

- `main.py` es un ejemplo de PyCharm y no es backend de producto.

## Producto y datos

- Vector (`concept-a`) es la interfaz canónica: `CANONICAL_UI=vector`.
- Horizon (`concept-b`) queda temporalmente como referencia no canónica y no recibirá paridad productiva.
- Los fixtures son sintéticos y deterministas: 8 expedientes y 20 señales con fecha fija.
- `MockOracleRepository` y `localStorage` son exclusivos de la demo.
- La abstracción de datos es parcial: `OracleProvider` y algunas pantallas todavía importan fixtures o almacenamiento directamente.
- La creación de expedientes usa `Date.now()` para el ID y la promoción de señal solo cambia su estado; ambas son limitaciones del prototipo.

## Dependencias y seguridad de suministro

- `package.json` declara todas las dependencias como `latest`; el lockfile fija las versiones actuales, pero regenerarlo sin control podría introducir saltos mayores.
- `npm ci` completó correctamente con 492 paquetes auditados.
- npm informa de 2 vulnerabilidades moderadas en la rama de Next/PostCSS. Su corrección automática propuesta implica un downgrade mayor inválido y no se ha aplicado.
- La instalación advierte de scripts pendientes de aprobación para paquetes transitivos. No se ha ampliado su autorización automáticamente.
- No se detectaron secretos en el repositorio durante la auditoría. Las credenciales facilitadas por el usuario no se copiaron a archivos ni se usaron.

## Riesgos inmediatos

1. La mayor parte del prototipo no está versionada y puede perderse con operaciones Git destructivas.
2. Toda la plataforma backend y de infraestructura está aún por construir.
3. El frontend está parcialmente acoplado a fixtures y `localStorage`.
4. Las declaraciones `latest` y la vulnerabilidad transitiva de PostCSS requieren una actualización controlada, no `npm audit fix --force`.
5. No existe todavía cobertura de auth, CSRF, multi-tenancy, RLS, API, workers, integraciones, backup o restore.
