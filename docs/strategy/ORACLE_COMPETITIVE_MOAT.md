**Síntesis:** nadie en España combina hoy licitaciones + entidades societarias + expediente con memoria + informes con evidencia auditada. La competencia real en cada venta será (1) el statu quo manual y (2) "ya tenemos alertas". INFERENCIA: la ventana de soledad competitiva es de 12-24 meses.

## 2. Ventajas defendibles: evaluación honesta

| Defensa candidata | ¿Real hoy? | Veredicto |
|---|---|---|
| **Memoria acumulada por expediente** (decisiones, hipótesis, feedback, evidencia) | HECHO técnica; se vuelve moat con uso | **La defensa principal a 2 años**: switching cost real — migrar 18 meses de memoria estratégica es impensable. Cada mes de uso la refuerza |
| **Evidencia y auditoría** (citas validadas, AIAuditLog, RLS) | HECHO | Diferenciador de confianza difícil de copiar rápido para un player de alertas; argumento de compra en regulados |
| **Fuentes españolas integradas** (PLACSP+BORME vía Signal) | HECHO | Ventaja de ejecución, no barrera absoluta (las fuentes son públicas); el valor está en la integración con el workflow |
| **Signal Avanza como capa propia** | HECHO (interno OPN) | Control del coste y del roadmap de fuentes; no dependencia de un proveedor de datos que pueda subir precios |
| **Feedback humano acumulado** (correcciones de triaje) | HECHO el registro; el reentrenamiento no | Moat futuro si se explota (mejores prioridades por cliente); no prometer aún |
| Taxonomías/plantillas por tipo de expediente | HECHO básico | Acelerador de onboarding, no moat |
| Playbooks, benchmarks entre clientes | No existe | P2: con ≥10 clientes, los benchmarks de adjudicación por sector serían un moat de datos real |
| Grafo de actores, workflows, integraciones | Parcial | Higiene competitiva, no defensa |
| UI bonita / "usamos IA" | — | **No son ventajas** (mercado: 79% de competidores invierten igual en GenAI; solo 23% cree tener ventaja sostenible — HECHO) |
| Especialización opcional por vertical | No aún | Palanca de pricing futura (plantilla sanidad/obra civil/energía), mantener core genérico |

## 3. Estrategia de moat recomendada (orden de inversión)

1. **Ahora (0-6 meses):** profundizar el ciclo memoria+evidencia — que cada piloto acumule decisiones e informes desde la semana 1 (el moat se planta en el onboarding). Coste: cero código, es proceso.
2. **6-12 meses:** explotar el feedback humano (prioridades aprendidas por cliente) y añadir la plantilla go/no-go de licitación — profundiza la cuña B2G.
3. **12+ meses (con ≥10 clientes):** benchmarks agregados de contratación por sector — el activo de datos que ningún entrante tendrá.
4. **Defensa contra el escenario Tendios-con-IA:** velocidad en la cuña + contratos anuales + memoria acumulada. Si ocurre, Oracle compite por arriba (expediente/decisión/evidencia), nunca por precio de alerta.
