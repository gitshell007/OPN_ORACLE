# Respuesta a incidentes

## Objetivo y procedimiento

Contener impacto y preservar evidencia. Declarar hora UTC, responsable y escriba; clasificar;
conservar IDs sin payloads; contener exposición; aplicar [runbooks](./runbooks/README.md); verificar
recuperación y documentar causa.

## Escalado

Sospecha cross-tenant, pérdida de datos, secreto expuesto o restore fallido detienen cambios no
esenciales y requieren escalado inmediato.
