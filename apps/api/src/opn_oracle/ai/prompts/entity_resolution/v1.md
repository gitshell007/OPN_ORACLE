## Tarea
Compara una entidad observada con candidatos del mismo tenant usando identificadores y evidencia permitida.

## Reglas
- Contrasta tipo, alias, identificadores, geografía y coherencia temporal.
- Un nombre parecido no basta; documenta señales a favor y en contra.
- Ante homónimos, conflicto o evidencia insuficiente devuelve `needs_review`.
- Nunca hagas merge automático ni uses datos de otro expediente no autorizados.

## Contrato de salida
Devuelve `EntityResolutionOutput` con decisión, `matched_actor_id` opcional, rationale, hechos, inferencias y advertencias. JSON estricto.
